Docker的安装与配置及镜像管理

发表于2022-06-29|更新于2022-06-29
|阅读量:

1.介绍

​ Docker 是一个开源的应用容器引擎，基于 Go语言 并遵从 Apache2.0 协议开源。

​ Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。

容器是完全使用沙箱机制，相互之间不会有任何接口,更重要的是容器性能开销极低。

Docker组件（核心）：镜像、容器、库。

采用C/S架构：客户端（执行程序）→通过命令行和API形式和守候程序（提供Docker服务）进行通讯。

  • 容器

  • 库

华云安harbor私服

2.优缺点

2.1 优点

  • 可移植性
  • 隔离性
  • 资源消耗较低
  • 快速部署

2.2 缺点

环境限制:

  • 必须在64位机器上运行，目前仅支持x86_64和AMD64位，不支持32位操作系统。
  • 系统的Linux内核必须是3.8或者更高,内核支持Device Mapper,AUFS,VFS,btrfs等存储格式。
  • 内核必须支持cgroups和命名空间。

4.配置私服镜像仓库

4.1 修改hosts文件

/etc/hosts文件中增加harbor私服的地址映射配置

4.2 拷贝harbor证书文件

  • 将harbor_certs.tgz(见文章末尾附件)文件拷贝到目录/etc/docker/目录下，执行解压操作

4.3 配置私服为可信任地址

  • 进入到/etc/docker/目录下，将"insecure-registries": ["harbor.huaun.com:11443"]增加进 daemon.json文件（若没有，则创建）中

4.4 配置连接密码

  • 创建目录
  • 将config.json文件拷贝进入到/root/.docker目录下，创建config.json文件 编辑内容 备注：config.json文件中 “auths”.”harbor.huaun.com:11443”.”auth”实际配置的是username:password 使用base64加密的结果

4.3 测试能否连接到华云安私服仓库

  • 从私服上拉取java8镜像进行测试

  • 查看镜像是否下载成功

    若出现java8镜像，则表示配置私服仓库成功

5.四大步

即我们使用Docker一般分为4个步骤：

  • build：制作编译一个镜像文件（image）
  • push：上传到远程仓库中
  • pull：从远程仓库拉取镜像文件
  • run：将镜像运行起来创建一个容器（container）

5.1 Docker Build

创建镜像的三种方式

  • 基于已有容器构建镜像主要是通过docker commit命令来构建新的镜像；
  • 基于本地模板导入；
  • 基于Dockerfile文件进行创建；
